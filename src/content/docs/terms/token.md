## 정의

**토큰(Token)** 은 LLM이 텍스트를 처리하는 최소 단위입니다. 단어보다 작은 조각(subword)으로, 영어 기준 1 토큰 ≈ 약 4글자, 한국어 기준 1 토큰 ≈ 약 1~2글자입니다. LLM의 비용과 컨텍스트 한계를 결정하는 핵심 단위입니다.

## 왜 중요한가

### 1. 비용 계산
API 과금은 토큰 단위입니다. 입력(프롬프트) 토큰 + 출력(생성) 토큰의 합산으로 과금됩니다.

### 2. 컨텍스트 윈도우
LLM이 한 번에 읽을 수 있는 최대 토큰 수가 정해져 있습니다. 예: Claude Sonnet은 200K 토큰. 이 한계 안에 프롬프트 + 참조 문서 + 생성 결과가 모두 들어가야 합니다.

### 3. 컨텍스트엔지니어링과의 관계
토큰 수 한계 때문에 "무엇을 보여주고 무엇을 빼는가"가 중요해집니다. 이것이 컨텍스트엔지니어링의 핵심 동기입니다.

## 토크나이저

텍스트를 토큰으로 변환하는 알고리즘입니다. 모델마다 토크나이저가 다르며, 같은 문장이라도 토큰 수가 다를 수 있습니다.

- **BPE (Byte Pair Encoding)**: GPT 시리즈, Claude 등 대부분의 LLM이 사용
- **SentencePiece**: 다국어 처리에 강한 토크나이저

### 한국어 토큰 비용
한국어는 영어 대비 같은 의미를 전달하는 데 **2~3배 토큰**을 소비합니다. API 비용과 컨텍스트 효율 모두 영향을 받으므로, 한국어 프로젝트에서는 프롬프트 최적화가 더 중요합니다.
