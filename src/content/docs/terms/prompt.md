## 정의

**프롬프트(Prompt)** 는 LLM에 보내는 입력 텍스트입니다. 질문, 지시, 맥락, 예시 등을 포함하여 AI의 응답 방향을 결정합니다. "무엇을 주느냐"가 "무엇을 받느냐"를 좌우합니다.

## 프롬프트의 구성 요소

### 1. System Prompt
AI의 역할, 어조, 제약을 정의합니다. 하네스 엔지니어링에서 `.cursorrules`가 이 역할을 합니다.

### 2. User Prompt
사용자의 실제 질문이나 지시입니다.

### 3. Context (맥락)
참조 문서, 이전 대화, 코드 등 AI가 판단에 필요한 배경 정보입니다.

### 4. Few-shot Examples
원하는 출력 형태를 예시로 보여주어 AI가 패턴을 따르게 합니다.

## 프롬프트 엔지니어링의 한계

프롬프트를 아무리 정교하게 다듬어도, **컨텍스트(AI가 무엇을 볼 수 있는가)** 가 부족하면 한계에 부딪힙니다. 이 한계를 구조적으로 돌파하는 것이 **컨텍스트엔지니어링** 이고, 그 실행 체계가 **하네스엔지니어링** 입니다.

| 관점 | 프롬프트 엔지니어링 | 컨텍스트 엔지니어링 |
|---|---|---|
| 초점 | "어떻게 물어보는가" | "무엇을 보여주는가" |
| 범위 | 단일 호출 | 시스템 전체 |
| 한계 | 표현의 한계 | 구조의 문제 → 구조로 해결 |
